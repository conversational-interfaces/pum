This specification defines the syntax and semantics for utility model's that are used to create and drive conversational experiences.

Introduction

Utility Model Format

Utility interface

This section is normative. 
        interface Utility {
          attribute Pairs preferences;
        };

preferences attribute

The preferences attribute specifies a Pairs object denoting a ranking configuration of all provided examples. It is specified in a key:value manner where key always takes on the value utility and value is a Config of pairings. 

          "utility": "[Pair]"

Pairs interface

        interface Pairs {
          attribute Config configs;
        };

configs attribute

The configs attribute specifies an tuple of Config objects. For the sake of simplicity we assume that the first Config object has higher utlity value than the second Config object. 

            "configs": "[firstConfig, secondConfig]"

Config interface

        interface Config {
          attribute String rankings;
        };

rankings attribute

The rankings attribute specifies a list containing any number of examples corresponding to desired outcomes. 

            [
              [{numWrongAttempts: 1, success: true}, {numWrongAttempts: 0, success: false}],
              [{numWrongAttempts: 5, success: true}, {numWrongAttempts: 3, success: false}],
            ]

Name

A name is a sequence of characters conforming to those in the XML standard. 

	                name ::= [ http://www.w3.org/TR/xml-names/#NT-NCName ]

String

A string literal represents a finite sequence of characters. It is surrounded by starting and ending quotation marks. 

	                string :: = '"' name '"'

Examples

This section is informative.

In the following example showcases an example where a higher number of wrong attempts has higher utility value the its corresponding paired object. 

        [
          [{numWrongAttempts: 1, success: true}, {numWrongAttempts: 0, success: false}],
          [{numWrongAttempts: 5, success: true}, {numWrongAttempts: 3, success: false}],
        ]

The following example showcases an instance where animating the Pig and Frog characters has higher utility than not when it comes to starting a game. 

        [
            [
              {
                "`is_pig_visible`": true,
                "`start_game`": true
              },
              {
                "`is_pig_visible`": false,
                "`start_game`": true
              }
            ],
            [
              {
                "`is_frog_visible`": true,
                "`start_game`": true
              },
              {
                "`is_frog_visible`": false,
                "`start_game`": true
              }
            ],
            [
              {
                "`is_fox_visible`": true,
                "`start_game`": true
              },
              {
                "`is_fox_visible`": false,
                "`start_game`": true
              }
            ]
        ]
